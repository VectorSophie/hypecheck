# Hypecheck v0.1.1: Real Manifest Parsing + Calibration

## Goal

Make agent-risk findings come from a repo's *actual* configuration, not regex over
its README. Today a bare README mention of "token"/".env" produces a HIGH
`secret-reference` finding, so almost every real tool trends `DANGEROUS` and the
verdict loses meaning. v0.1.1 parses `.claude-plugin/plugin.json`, `hooks/hooks.json`,
and `.mcp.json` from GitHub candidates, derives precise findings from them, and
recalibrates the README heuristics down to corroboration.

Approach: **A (parse-and-supersede) with a hint of B** — manifest-derived findings
drive the verdict; the README-regex findings are kept but demoted to low when a
manifest confirms/covers them, so evidence is never deleted.

## Non-goals (this release)

- npm tarball inspection (npm candidates keep using `package.json` + README) — v0.2.x.
- Following arbitrary `"hooks": "./custom/path"` references — only the two
  conventional locations + inline `plugin.json` are read.
- `compare` / `explain` commands — v0.2.0.

## Fetching (`fetchers.js`)

Extend `fetchGithubCandidate` with the existing best-effort contents-API pattern
(base64 decode, `JSON.parse`, `null` on miss) for:

- `.claude-plugin/plugin.json`
- `hooks/hooks.json`
- `.mcp.json`

Attach `data.manifests = { plugin, hooks, mcp }` (each `null` when absent). Adds up
to 3 GitHub API calls per eval, raising the value of `GITHUB_TOKEN`.

## Extraction (`extractors.js`)

Pure normalizers (no I/O):

```js
extractHookEvents(manifests) -> [{ event, command }]
// merge plugin.json.hooks (inline object) + hooks.json; each event's entries
// flattened to { event, command } pairs.

extractMcpServers(manifests) -> [{ name, command, needsSecrets }]
// from .mcp.json.mcpServers + plugin.json.mcpServers; needsSecrets = an
// env/headers block exists (presence only — values are never read).
```

## Analysis (`analyze.js`)

`analyzeManifests(data, findings)`:

- **Configured hook** per event, severity by sensitivity:
  - high: `PreToolUse`, `PostToolUse`, `PostToolUseFailure`, `UserPromptSubmit`
  - medium: `SessionStart`, `Stop`, `SubagentStop`, `Setup`, `Notification`
  - id `configured-hook`; evidence names the event + truncated command.
- **Shell in hook**: if a hook command matches `curl ... | sh`, `| sh`, `| bash`,
  or `base64 -d` → an additional high `shell-in-hook` finding (real exec evidence).
- **MCP servers**: one aggregate `mcp-servers` finding — server count + whether any
  `needsSecrets`; severity low (medium if 5+ servers → budget pressure).

**Supersede + hint of B:** pass `manifestHooksFound` into `analyzeText`. When true,
the README-regex `dangerous-hook-event` and `shell-capability-mentioned` are emitted
at **low** severity ("mentioned in README; see configured-hook findings") instead of
their current medium/high.

**Calibration:** `secret-reference` severity moves **high → low** unconditionally. A
README mentioning credentials is a heads-up, not a danger. HIGH risk now comes from
configured behavior (lifecycle scripts, configured hooks, shell-in-hook). This keeps
`sketchy-agent` (postinstall + shell-exec dep = 2 HIGH security) at `DANGEROUS`.

## DX / fixes

- `bin/hypecheck.js`: `try { process.loadEnvFile(); } catch {}` at startup (Node
  >=20.12; no-op otherwise) so `GITHUB_TOKEN` in a local `.env` is picked up without
  `--env-file`.
- Version bump `0.1.0` -> `0.1.1` in `package.json` and `claude-plugin/.claude-plugin/plugin.json`.

## Test plan (node:test, injected fetchImpl)

- `fetchers.test.js`: GitHub fetch attaches `manifests`; missing manifest files
  tolerated (all three `null`).
- `extractors.test.js`: hook events from inline `plugin.json.hooks` AND `hooks.json`;
  MCP server extraction; `needsSecrets` true when `env` present, and no secret value
  appears in the output.
- `analyze.test.js`: high vs medium hook-event tiers; `shell-in-hook` on `curl|sh`;
  README `dangerous-hook-event` demoted to low when a hooks manifest is present;
  `secret-reference` now low.
- `score.test.js`: a single configured `PreToolUse` hook can reach `DANGEROUS`; a bare
  README secret mention alone cannot.

## Acceptance

- Re-running the live Context7 / claude-obsidian evals no longer reaches `DANGEROUS`
  solely on a README secret mention.
- A repo that actually commits a `PreToolUse` hook running a shell command is
  `DANGEROUS` with evidence quoting the hook.
